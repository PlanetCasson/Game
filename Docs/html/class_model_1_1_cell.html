Detailed Description

Quad-Edge data structure representing the graph iteself.

The Cell is a Quad-Edge structure that contains a list of edges, verticies, and faces. It is capable of representing all surfaces that are locally homeomorphic to the plane. It is encoded in such a way that it contains both the graph itself and its dual graph. For more details on how this encoding is carried out, see Edge.

Benefits of this data structure is the easy and fast access of locality data and the O(1) CW and CCW traversal around points/faces. This second point will be extremely important in our game about CCW traversal on graphs embedded in surfaces.

void Model.Cell.killFaceEdge ( Face  f,
Vertex  orig,
Vertex  dest

The method of reducing the quad-edge structure. killFaceEdge does the opposite of makeFaceEdge and "rejoins" a subdivided face by deleting the face f and transfers its edges to the face to the left of the edge running from orig to dest.

This first finds a list of edges that needs to be moved to the other face, and calls Edge.RejoinFaceVertex with the appropriate parameters.

TO utilize RejoinFaceVertex, we pass in arguments to indicate rejoining a vertex corresponding to the face f in the dual graph.

If the specified face does not contain an edge running from the orig to dest that also has another face to its right, then an ArgumentException error will be thrown.

Note, this might create digons.

Parameters
fThe face to be deleted.
origThe orig of the edge that currently divides the face.
destThe dest of the edge that currently divides the face.

The method of reducing the quad-edge structure. killVertexEdge does the opposite of makeVertexEdge and "unsplits" a vertex by deleting the vertex v and transfers its edges to the other vertex shared between the left and right faces.

This first finds a list of edges that needs to be moved to the other vertex, and calls Edge.RejoinFaceVertex with the appropriate parameters.

If the specified left and right faces does not share 2 verticies where one of them is the vertex v, then an ArgumentException error will be thrown.

Note, this might create digons.

Parameters
vThe vertex to be deleted.
leftThe face to the left of the vertex.
rightThe face to the right of the vertex.

The method of expanding from the primitive quad-edge structure. makeFaceEdge subdivides the face "f" into two along an edge that points from orig to dest.

This first finds a list of edges that needs to be moved to surround the new face, creates a new, unlinked face, and calls Edge.SplitFaceVertex with the appropriate parameters.

To utilize SplitFaceVertex, we pass in arguments to indicate splitting a vertex corresponding to the face f in the dual graph.

If the specified face does not contain the verticies orig and dest, an ArgumentException error will be thrown.(uncatched from SplitFaceVertex).

The method of expanding from the primitive quad-edge structure. makeVertexEdge splits the vertex "v" into two and connect it with an edge whose left face is "left" and right face is "right".

This first finds a list of edges that needs to be moved to the new vertex, creates a new, unlinked vertex, and calls Edge.SplitFaceVertex with the appropriate parameters.

If the specified left and right face does not share a vertex, an ArgumentException error will be thrown.(uncatched from SplitFaceVertex).
